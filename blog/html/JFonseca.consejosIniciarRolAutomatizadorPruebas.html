Resumen

No pocos egresados de la carrera de computacin e informtica se dedican a realizar pruebas de software para aplicaciones mviles y servicios o sitios web. En ciertas ocasiones, muchos de ellos no se sienten a gusto ejecutando simplemente pruebas manuales y optan por aplicar a puestos de automatizacin donde tienen una oportunidad para aplicar sus habilidades de programacin. Sin embargo, para un principiante, hay gajes del oficio que no siempre se explican en clase y que es necesario dominar para evitar malas experiencias. En este notebook nos propusimos recopilar algunos consejos con base en la observaciones propias y recomendaciones de compaeros con ms experiencia (a algunos de ellos lamentablemente les cost su puesto). Se deja como ejercicio del lector evaluar si estos conocimientos son tiles.

Locators

A menos que se est usando Sikuli o alguna aplicacin de visin por computadora el uso de localizadores XPath o CSS para interactuar y obtener el estado de elemento del Modelo de Objeto del Document (DOM, por sus siglas en ingls) es inevitable. La manera ms sencilla de obtener un localizador es generarlo automticamente desde la consola de desarrollo del navegador web (click derecho sobre la etiqueta > copiar > XPATH o CSS), y el resultado para el encabezado de la columna "ESC/ESH*" de abajo podra ser algo monstruoso como /html/body/div[1]/div/div[1]/div[4]/div[2]/div/div[2]/div[2]/table/tbody/tr[1]/th[6]/div/span en XPATH, y th.dash-header:nth-child(6) > div:nth-child(1) > span:nth-child(1) en CSS.

Adems de ser un ruta complicada y difcil de escribir, suele pasar que los desarrolladores cambian continuamente el layout de la pgina web para completar sus historias de usuario, haciendo que los localizadores usados en pruebas anteriores ya no funcionen, lo que se traduce en una bola de nieve de trabajo para futuras iteraiones. Esto es particularmente peligros en aplicaciones que sirven dinmicamente el contenido web, como es el caso de ReactJS. Para mitigarlo, el autor Sergii Korol explica en su Gua para Automatizar Aplicaciones React que no se debe aceptar historias de usuario sin tareas para que los desarrolladores asignen IDs a las etiquetas ms importantes del DOM y que se debe usar Javascript para interactuar con componentes complicados como DatePickers o reproductores web (por ejemplo, driver.execute_script('document.getElementById("...").play()')).

Para el caso del localizador del encabezado de la tabla de arriba, una vez agregado el ID, el resultado se ve as: //th[contains(@class, 'dash-header') and @data-dash-column="ESC/ESH*"].
