Interpolación de banda limitada para el tipo de cambio del dólar¶

Por Juan M. Fonseca-Solís, Mar 2015

Resumen¶

La reconstrucción de secuencias temporales continuas a partir de muestras tomadas discretamente es un problema que ha sido resuelto tradicionalmente usando el teorema de Nyquist. Este teorema establece que la frecuencia de muestreo debe ser el doble de la frecuencia máxima contenida en la señal original para evitar la pérdida de información (ver teorema de Nyquist aquí). La reconstrucción –también llamada interpolación– se logra convolucionando una función de interpolación –también llamada núcleo de interpolación– con cada muestra disponible, luego se realiza una suma de todas las convoluciones para obtener la función continua. Hasta aquí, teoría conocida. El problema se vuelve interesante cuando se quiere reconstruir una señal a partir de muestras tomadas aleatoriamente, es decir, donde no hay una frecuencia de muestreo fija.1 En este caso, si las frecuencias altas son despreciables, se puede usar el método de interpolación de banda limitada (BLI, por sus siglas en inglés) que utiliza las series de Fourier y el método de mínimos cuadrados para obtener una aproximación de la señal original. En este notebook se explica con mayor detalle en qué consiste el método mencionado y se presenta un ejemplo estudiando el tipo de cambio del dólar en Costa Rica ofrecido por el Banco Central de Costa Rica (BCCR).

Interpolación de banda limitada¶

Una señal discreta de $N$ entradas $\{(x_i,z_i)\}_{i=0}^{N-1}$, con soporte temporal $0<x_i \le T$, se puede modelar como un sistema lineal así [2]:2

$$ Fc=z,\qquad (1) $$

donde $\{F\}_{ik}=e^{j2\pi \frac{x_ik}{T}}$ es la $k$-ésima entrada del $i$-ésimo vector ortogonal de las series de Fourier y $c=(c_{-M}, \ldots, c_M)^t$ es el vector de coeficientes usados para las tareas de análisis y reconstrucción. Suponiendo que la cantidad de filas y columnas de $F$ es el mismo (esto es, $N=2M+1$), se podría resolver el sistema encontrando la inversa de la matriz, de la siguiente manera:

$$ c=F^{-1}z, $$

Este sería el caso ideal. Sin embargo, en la realidad esto no pasa porque la cantidad de muestras de la señal usualmente es mayor al número de coeficientes elegido para el modelado ($2M+1<N$). Lo que obliga a realizar una aproximación, por ejemplo, usando mínimos cuadrados:3

$$ \hat{c}=(F^t F)^{-1}F^tz. $$

Interpolación¶

El procedimiento anterior modela la señal en los puntos que fueron muestreados originalmente, pero no en medio de las muestras, donde es necesaria realizar la interpolación. Por lo que es necesario realizar un remuestreo lineal en el intervalo $[0, T]$, para ello se puede usar una simple regla de tres:

$$ \tilde{x}_i=i\frac{T}{L}, $$

y sustituir $x_i$ por el $\tilde{x}_i$ en la ecuación 1. El sistema resultante es modela por Prandoni y Veterli en la siguiente expresión:

$$ \tilde{z}_i=\tilde{F}\hat{c}. $$

Interpolación de banda limitada¶

Ahora usamos BLI para aproximar la señal original usando una curva más suave.

In [20]:
def BL_interp_1D(x, z, T, order, grid_step=0.01, win=True):
    """
    Band-limited interpolation of 1D functions (tomado de P.Prandoni y M.Vetterli)
    """
    # Create Fourier order vector
    k = np.expand_dims(arange(-order, order+1), 0)

    # construct the Fourier matrix
    F = exp(2j*pi*x*k/(T[1]-T[0]))

    # Least-square projection (alternatively linalg.lstsq can be used)
    C = dot(dot(linalg.inv(dot(F.T,F)), F.T), z)

    # create new evenly spaced grid
    xg = expand_dims(arange(T[0], T[1], grid_step), 1)

    # window the Fourier coefficients if requested
    if (win):
        C *= expand_dims(hanning(2*order+1), 1)

    zg = dot(exp(2j*pi*xg*k/(T[1]-T[0])), C)

    return zg, xg, C, k

'''
Aplicamos la interpolación BL
'''
orden = 20 # número de coeficientes de la aproximación (empírico)
T = [0, N]
xmp = expand_dims(array(xm),1)
zmp = expand_dims(array(zm),1)

# calculamos la interpolación BL
zp1, xp1, C1, k1 = BL_interp_1D(xmp, zmp, T, orden, win=False)

Cantidad de coeficientes¶

Como punto adicional, se pueden estudiar las series de Fourier como método de "compresión". Como la señal analizada en el ejemplo es real, los curva de coeficientes es simétrica (una propiedad de las series de Fourier), y por lo tanto no necesitamos almacenar los $2M+1$ coeficientes para realizar la reconstrucción. Recordando que $N$ es la cantidad de muestras de la señal original y $M$ la cantidad de coeficientes usados, esto nos permite calcular el "porcentaje de compresión" $c$ como sigue:

$$c=\frac{N-M}{N}.$$
In [17]:
c = float(N-orden)/N
print 'Muestras tomadas: %i' % N
print 'Cantidad de coeficientes: %i (necesarios para reproducir la señal)' % orden
